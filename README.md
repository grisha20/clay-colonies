# Clayfolk

Clayfolk — короткая кооперативная colony-sim песочница про маленький глиняный народец.

Игроки не управляют каждым человечком напрямую. Они ставят задачи, рисуют зоны, выбирают постройки, выставляют приоритеты и смотрят, как жители сами добывают глину, дерево, камень, еду и воду, строят поселение, чинят стены, лечат раненых, собираются в отряды и защищаются от угроз.

Цель партии — за 2-3 часа пройти путь от кучки глиняных жителей у костра до организованного племени, пережить хаос и дойти до финального события: осады, ритуала, набега, катастрофы или босса.

## Текущий код

Проект вырос из прототипа наблюдательной симуляции. Поэтому в коде пока остаются старые технические имена:

- `Ant`, `Colony`, `queen`, `brood`, `pheromone`, `colonyId`;
- `server/src/sim/ant/*`;
- `antPaths`, `world.colonies[]`, `spider_genome.json`;
- HUD-метрики и команды вроде `bench:server` на `ants`.

Пока код не переименован, документация должна ссылаться на эти имена точно. В человеческом описании проект уже считается Clayfolk.

## Текущий стек

- `server` — Node.js + TypeScript, авторитетная симуляция.
- `client` — Vite + TypeScript + PixiJS, рендер снапшотов.
- `shared` — общие типы снапшота мира.
- `snapshot.json` — локальное сохранение мира.

## Команды

## Что важно сейчас

Новая цель — не бесконечно развивать старую наблюдательную модель, а постепенно повернуть проект к партии на 2-3 часа:

```text
маленькая группа -> костёр -> сбор руками (ветки, камешки, глина) -> мастерская -> топоры и кирки -> рубка леса и добыча камня -> склад -> постройки -> роли -> стены/оборона -> атака или событие -> восстановление -> финал партии
```

Код менять нужно малыми шагами, по правилам:

- после каждого серверного изменения — `npm run check` и `npm run check:slice`;
- смена `shared/types.ts` = поднять версии снапшота/протокола = Сброс (штатно);
- баланс и поведение смотреть с чистого мира (Сброс.bat);
- маленькие коммиты, main всегда рабочий, незакоммиченного не копить;
- чеклисты в `docs/` вести в том же коммите, что и код.
