# План: от текущего вида к целевому (по двум скриншотам)

> 03.07.2026, Fable 5. Слева в сравнении — текущая игра (ветка `fable5-dev`),
> справа — макет-мечта, сгенерированный ChatGPT. Макет — это ориентир по
> композиции и настроению, а не техзадание пиксель-в-пиксель.
> Порядок этапов — от простого к сложному; каждый этап самостоятельно ценен
> и оставляет игру рабочей.

## 1. Что именно отличается (разбор разрыва)

**Мир и арт:**
- У нас: ровная зелёная трава, пустоватая карта, еда — зелёные крошки, глина —
  мелкие комья, пруд — плоский синий эллипс, человечки мелкие и одинаковые.
- В цели: тёплая песочная поляна-лагерь, костёр с тотемом в центре, ресурсы —
  крупные читаемые объекты (ягодные кусты, рыжие глыбы глины, кучи палок, серые
  камни), у пруда органичный берег, человечки крупные и РАЗНЫЕ: с копьями,
  кирками, корзинами, палками на плече.

**Интерфейс:**
- У нас: тёмные панели, HUD-таблица справа, инструменты строкой сверху.
- В цели: пергаментно-деревянная тема; ресурс-бар сверху по центру с иконками
  и приростом (+12); панель задач слева; меню построек карточками с ценами
  внизу; панель выбранного жителя слева-внизу; минимапа справа-внизу;
  пауза/ускорение/настройки справа-вверху.

**Механики, которых у нас ещё нет:**
- камень как ресурс (у нас глина+дерево+еда);
- видимые роли (воины с копьями и т.д.) — это Фаза 5 основного плана;
- задачи партии (Gather Clay 152/200, Build Hut 0/2) — это росток Фазы 7;
- выбор жителя кликом; минимапа; пауза; мораль (смайлик 36).

**Что из макета НЕ копируем буквально (решено не гнаться):**
- рыбак с удочкой, посадка кустов, «настроение 36» — поздний контент;
- цены в макете (Hut 20 палок) — у нас свой баланс (8 глины + 5 дерева);
- кактусы — не наш биом; чужие ассеты не тащим, всё рисуем своими
  пиксельными спрайтами (`client/src/sprites.ts`) и Graphics.

## Этап 1. Тёплый мир и читаемые ресурсы (клиент, Степан) — самое простое

### 1.1 Палитра «песочницы»
- **Что:** сблизить тона с макетом: трава теплее/светлее (`tile.tint` в
  `ground.ts`, сейчас `0xa0b85c`), песчаная поляна светлее и желтее
  (двухтоновый песок), общий тёплый кант.
- **Файлы:** `client/src/render/surface/ground.ts` (тинты, цвета флеков).
- **Готово когда:** скриншот рядом с макетом — «одна семья» по цвету.

### 1.2 Еда = ягодный куст
- **Что:** источник еды рисовать как куст (зелёная крона + красные ягоды,
  количество ягод от `amount`), а не россыпь зелёных крошек. Куст уже почти
  есть в `drawBush(…, berries=true)` — сделать спрайт-версию для пула.
- **Файлы:** `client/src/sprites.ts` (спрайт куста и ягод),
  `client/src/render/surface/entities.ts` (`updateSurfaceFood`).
- **Риск:** сервер не меняем — «куст» это визуал того же `FoodSource`.
- **Готово когда:** с обычного зума сразу видно «вон ягоды», пустеющий куст
  теряет ягоды.

### 1.3 Глина и дерево — крупные объекты
- **Что:** узел глины — 2-3 крупные рыжие глыбы со светлым верхом (как в
  макете слева), дерево — заметная куча палок; у лагеря кучи уже есть.
- **Файлы:** `client/src/sprites.ts`, `entities.ts` (`updateSurfaceResources`).
- **Готово когда:** узлы не путаются с декором и заметны с любого зума.

### 1.4 Центр лагеря: костёр + тотем + кольцо камней
- **Что:** пересобрать композицию лагеря в `entrance.ts`: костёр по центру
  утоптанного круга, тотем-идол рядом, кольцо камушков, кучи ресурсов вокруг
  по дуге. Мерцание костра уже есть — усилить языками пламени (2-3 кадра).
- **Файлы:** `client/src/render/surface/entrance.ts`, `scene.ts` (fireGlow).
- **Готово когда:** лагерь читается как «сердце» кадра, как в макете.

### 1.5 Пруд с берегом
- **Что:** давно запланированный шаг 1.1 старого плана: маска с волнами,
  песчаная кайма, пиксельная бахрома, блики (метод из
  `docs/Метод переходов поверхности.md`).
- **Файлы:** `ground.ts` (`drawPond`, `isWaterPatch`).
- **Готово когда:** пруд как в макете — с живым берегом, не «наклейка».

## Этап 2. UI-каркас в пергаментной теме (клиент)

### 2.1 Тема панелей
- **Что:** переключить CSS в `main.ts`: бежевый фон (#f0e2c0…), коричневые
  рамки 2-3px со скруглением, тёмно-коричневый текст. Все панели разом —
  это только стили.
- **Файлы:** `client/src/main.ts` (блок `style.textContent`).
- **Риск:** нет. Самая дешёвая «половина пути» к макету.

### 2.2 Ресурс-бар сверху по центру
- **Что:** заменить таблицу племени A на бар: [иконка глины] 26 (+12)
  [ягоды/еда] 1582 [палки] 32 [камень —] [жители] 31/23. Иконки — мелкие
  canvas-спрайты из `sprites.ts` (data-URL в `<img>`).
  Прирост считает КЛИЕНТ: разница запаса за последние ~60 сек по снапшотам.
- **Файлы:** `main.ts` (разметка + `updateHud`), `sprites.ts` (экспорт
  иконок как dataURL).
- **Не трогаем:** сервер и снапшот.
- **Готово когда:** цифры и «+прирост» живые; племя B — компактно или по
  тумблеру (мы играем за A).

### 2.3 Меню построек карточками внизу
- **Что:** перенести «Хижина/Стена» из верхней строки в нижний ряд карточек:
  иконка, название, цена (8 глины + 5 дерева / 2 глины). Карточка серая,
  если ресурсов не хватает (клиент знает запасы из снапшота). «Зоны/Ластик/
  Еда» остаются инструментами сверху или переезжают тоже — по вкусу.
- **Файлы:** `main.ts`.
- **Готово когда:** как в макете: выбор постройки = клик по карточке, цена
  видна до клика.

### 2.4 Пауза
- **Что:** кнопка ⏸ рядом со скоростями. Сервер: разрешить `setSpeed 0`
  (сейчас `Math.max(1, …)` в `index.ts`; в `loop.ts` при 0 — не звать `step`,
  но продолжать broadcast).
- **Файлы:** `client/src/main.ts`, `server/src/index.ts`, `server/src/loop.ts`.
- **Риск:** маленький серверный шаг; проверить, что автосейв на паузе не
  спамит и клиент не считает паузу разрывом.
- **Готово когда:** пауза останавливает мир, HUD живой, снятие паузы — без
  скачка.

## Этап 3. Минимапа и выбор жителя (клиент, посложнее)

### 3.1 Минимапа
- **Что:** маленький canvas/RenderTexture справа-внизу: фон карты (упрощённо:
  трава/поляны/пруды) рисуется один раз; поверх каждый снапшот — точки
  жителей (по цвету племени), паук, постройки; рамка текущего вьюпорта.
  Клик по минимапе = перенос камеры.
- **Файлы:** `main.ts` (панель), новый `client/src/render/minimap.ts`.
- **Риск:** производительность — точек мало (≤60), дёшево.
- **Готово когда:** по минимапе видно всю картину и можно прыгать камерой.

### 3.2 Выбор жителя
- **Что:** клик инструментом «Еда» (или отдельный режим «Выбор») по жителю —
  панель слева-внизу: портрет (спрайт крупно), занятие по-русски
  (`job`/`state` → «Несёт глину», «Строит», «Разведка»), энергия как полоска.
  Подсветка выбранного кольцом. Кнопок приказа юниту НЕ делаем — управление
  остаётся непрямым (принцип игры).
- **Файлы:** `main.ts` (панель, picking по `latestWorld.ants` в радиусе
  клика), `render/surface/entities.ts` (кольцо выбора).
- **Не трогаем:** сервер — id жителя уже есть в снапшоте.
- **Готово когда:** тыкаешь в толпу — видишь, кто это и чем занят; панель
  очищается, если житель умер/пропал.

## Этап 4. Камень + осмысленные постройки (сервер Гриша + клиент)

### 4.1 Ресурс «камень»
- **Что:** третий ресурс по готовому образцу глины/дерева: `kind: "stone"` в
  `ResourceNode`, `colony.stone`, цель запаса, сборщики (economy уже умеет),
  серые кучи у лагеря, иконка в ресурс-баре. Снапшот v5.
- **Файлы:** `shared/types.ts` (+bump версий), `server/src/sim/world.ts`,
  `economy.ts`, `config.ts`; клиент: `sprites.ts`, `entities.ts`,
  `entrance.ts`, `main.ts`.
- **Риск:** копия уже отлаженного пути — низкий. **Сброс мира обязателен.**
- **Готово когда:** `check:slice` дополнен камнем и проходит; на глаз — серые
  кучи растут у лагеря.

### 4.2 Новые постройки с понятной пользой
- **Что:** к хижине и стене добавить 1-2 постройки из макета, у каждой —
  реальный эффект, не декор:
  - **Склад** (дерево+камень): вторая точка сдачи ресурсов — носильщики
    сдают в ближайшую (вход или склад), пробеги короче;
  - **Костровище/факел** (камень+дерево): расширяет «домашний» радиус
    (задел под свет/оборону Фазы 6).
- **Файлы:** `sim/building.ts` (типы/цены), `ant/movement.ts` (`tryCrossLayer`
  → выбор ближайшей точки сдачи), `ant/build.ts`, клиентские карточки.
- **Риск:** сдача в склад трогает горячий путь фуражировки — делать вторым
  шагом, отдельно, с прогоном `check:slice`.
- **Готово когда:** склад у дальнего узла глины сокращает маршрут — видно
  глазами, добыча/мин растёт.

## Этап 5. Видимые роли и «живость» (Фаза 5 основного плана)

### 5.1 Роли с предметами
- **Что:** явный `role: worker | builder | guard` на сервере (распределение по
  образцу `assignForageRoles`), у каждого — предмет на спрайте: копьё у
  охранника (стоят у лагеря/ворот, как в макете справа), молоточек у
  строителя, кирка у сборщика камня/глины, корзина у носильщика ягод,
  палка на плече у носильщика дерева.
- **Файлы:** сервер: `shared/types.ts`, `economy.ts`/новый распределитель;
  клиент: `sprites.ts` (варианты спрайтов), `entities.ts` (выбор текстуры по
  роли и грузу).
- **Риск:** много вариантов спрайтов — рисовать пакетно; серверная часть
  проста. Сброс мира.
- **Готово когда:** «с первого взгляда понятно, кто несёт камень, кто строит,
  кто держит линию» — критерий прямо из Основной мысли.

### 5.2 Микро-анимации
- **Что:** 2-кадровые анимации через смену текстуры по тику: удар киркой у
  узла, стук у стройки, покачивание груза. Без скелетов и спрайт-листов.
- **Файлы:** `sprites.ts`, `entities.ts`.
- **Готово когда:** толпа «суетится», а не скользит.

## Этап 6. Панель задач партии (мостик к Фазе 7)

- **Файлы:** `shared/types.ts` (+bump), новый `server/src/sim/objectives.ts`,
  `sim/step.ts` (обновление прогресса), клиент `main.ts`.
- **Риск:** низкий технически; главный вопрос — какие цели весёлые. Начать с
  3 зашитых, тюнить по ощущениям.
- **Готово когда:** новичок открывает игру и панель задач сама ведёт его по
  петле: добудь → построй → пережди угрозу.

## Кто что делает: Fable 5 и Codex

Исполнителей два: **Fable 5** (я) и **Codex** (запускает Степан локально).
Разница простая: Codex работает на машине Степана — Степан сразу ВИДИТ
результат на экране и может гонять короткие итерации «поправь цвет — смотрю».
Я экрана не вижу, зато надёжно везу многофайловые серверные механики и
проверяю их по поведению headless-прогонами.

**Codex (визуал и всё, что проверяется глазами; промты — в конце документа):**
- Весь этап 1: палитра (1.1), ягодные кусты (1.2), глыбы глины/палки (1.3),
  композиция лагеря с тотемом (1.4), берег пруда (1.5).
- 2.1 Пергаментная тема, 2.3 вид карточек построек.
- 5.1 спрайты ролей (копьё/кирка/корзина) и 5.2 микро-анимации.
- Правки-полировки моих UI-черновиков по замечаниям Степана.
- Формат работы: один шаг = один промт из этого документа, после — Степан
  смотрит на экран и либо принимает, либо даёт Codex правку.

**Fable 5 (механики, протокол, сквозные изменения, поведенческие проверки):**
- 2.4 Пауза (сервер + кнопка).
- 2.2 Ресурс-бар — логика (данные, прирост по снапшотам); стиль потом Codex.
- 3.1 Минимапа и 3.2 выбор жителя — рабочий каркас (данные, клики, обновление).
- Весь этап 4: камень (снапшот v5) и склад как вторая точка сдачи.
- 5.1 серверная часть ролей (распределение, поведение охранника).
- Этап 6: задачи партии (objectives.ts + черновик панели).
- Всё, что меняет `shared/types.ts`, версии снапшота/протокола и `sim/` —
  только я: такие изменения требуют сквозной аккуратности и headless-проверок
  (`npm run check:slice` дополняю на каждом шаге).

**Правила, чтобы не столкнуться:**
1. Codex не трогает `server/src/sim/`, `shared/types.ts` и версии снапшота —
   это моя зона. Я не трогаю файл, который Codex прямо сейчас полирует.
2. Порядок удобен такой: Codex идёт по этапам 1 → 2.1 → 2.3, я параллельно
   2.4 → 4 → 6; встречаемся на этапе 5 (я сервер, Codex спрайты).
3. Каждый шаг Codex — маленький коммит после «глазами ок» от Степана.
   Каждый мой шаг — маленький коммит после `npm run check` + headless-прогона.
4. Оба работаем в `fable5-dev`.

## Напоминания
- **Каждое изменение снапшота (этапы 4, 5, 6) = bump версии + Сброс.bat.**
- Чисто клиентские этапы (1-3) мир не сбрасывают.
- После каждого этапа: `npm run check`, `npm run check:slice`, глазами 10
  минут на 1x и 20x.
- Порядок 1→2→3 можно делать параллельно с серверными 4→5→6 (Степан/Гриша).
