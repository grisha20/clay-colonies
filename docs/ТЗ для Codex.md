## Контекст (кратко)

Clayfolk — colony-sim песочница: сервер (Node+TS) считает мир, клиент (Vite+PixiJS)
только рисует снапшоты и шлёт команды. Мы ведём игру к виду из макета: тёплая
песочница, читаемые ресурсы, пергаментный UI. Механики уже готовы (ресурсы
глина/дерево/камень, зоны, постройки хижина/склад/стена, стража, задачи партии,
ресурс-бар, минимапа, выбор жителя) — твоя работа почти целиком ВИЗУАЛ.

## Правила (нарушение = задание не принято)

1. **Работай только в клиенте**: `client/src/**`. Разрешено также читать (не менять)
   `shared/types.ts` — чтобы знать поля снапшота.
2. **ЗАПРЕЩЕНО менять**: `server/src/**`, `shared/types.ts`, версии снапшота/протокола,
   форматы сетевых команд. Поведение симуляции от твоих правок меняться не должно.
3. **Ассеты**: только пиксельные спрайты в `client/src/sprites.ts` (рисуются строками)
   и паки, которые УЖЕ лежат в `client/public/assets/` (summer-plains и др.).
   Ничего не скачивать, чужие паки не добавлять.
4. После каждого задания: `npm run check` должен проходить; игра запускается
   (`npm run dev`), FPS не проседает заметно (метрика Render ms в нижней панели).
5. Один коммит на задание, сообщение: «Codex З<N>: <суть>». Ветка `fable5-dev`.
6. Технические имена (`Ant`, `Colony`, `ant`, `antPool`) не переименовывать.
7. Если для красоты «нужна» правка сервера — НЕ делай её; запиши предложение
   комментарием `// TODO(Fable5): ...` и продолжай без неё.

## Полезные факты по коду

- Тайл поверхности 8px (`SURFACE_TILE_SIZE`), карта 480×480.
- Земля/декор: `client/src/render/surface/ground.ts` (трава — спрайты с
  `tile.tint = 0xa0b85c`; поляны лагерей; `drawPond`; `drawBush(…, berries)` уже есть).
- Сущности: `client/src/render/surface/entities.ts` (еда `updateSurfaceFood`, ресурсы
  `updateSurfaceResources`, жители `updateSurfaceAnts` — там же временная «копьё-черта»
  у стражи и кольцо выбора).
- Лагерь: `client/src/render/surface/entrance.ts` (костёр, две фигуры, кучи еды/глины/
  дерева/камня); мерцание — `updateFireGlow` в `scene.ts`.
- Постройки рисуются в `scene.ts` → `updateBuildings` (Graphics-заглушки).
- Спрайты: `client/src/sprites.ts` — пиксельные карты строками + палитры; иконки для
  DOM — `spriteIconDataUrl`.
- UI: весь HTML/CSS в `client/src/main.ts` (`appRoot.innerHTML` + `style.textContent`).
  Панели: `#resource-bar`, `.tasksPanel`, `#unit-panel`, `.minimapPanel`, `.status`.
- Данные для UI: `latestWorld.colonies[0].colony` — `food/clay/wood/stone`,
  `population.workers`, `nestCapacity`; `world.objectives`; у жителя — `job`
  (`forage|harvest|build|guard|idle`), `forageRole`, `carryKind`, `carrying`, `energy`.
- Цены построек (зашиты на сервере, на клиенте дублируй константой с комментарием):
  хижина = 8 глины + 5 дерева; склад = 6 дерева + 4 камня; стена = 2 глины за сегмент.

## Задания (строго по порядку)

### З1. Тёплая палитра мира
`ground.ts`: подобрать тона ближе к макету — трава теплее/светлее (сейчас tint
`0xa0b85c`), песчаные поляны лагерей светлее и желтее, флеки каймы в тон.
**Приёмка:** скриншот игры и макет выглядят «одной семьёй» по цвету; читаемость
жителей на траве и песке не ухудшилась.

### З2. Еда = ягодный куст
`sprites.ts` + `entities.ts` (`updateSurfaceFood`): источник еды рисовать как куст
(тёмно/светло-зелёная крона) с красными ягодами; ягод = `ceil(amount/15)`, максимум
8; при amount→0 куст пустой (без ягод). Убрать нынешнюю россыпь зелёных крошек.
Тень оставить (updateSurfaceShadows не трогать).
**Приёмка:** с дефолтного зума мгновенно видно «ягоды»; пустеющий куст лысеет.

### З3. Крупные узлы ресурсов
`sprites.ts` + `entities.ts` (`updateSurfaceResources`): глина — 2-3 крупные рыжие
глыбы со светлым верхом; дерево — заметная кучка палок; камень — серые валуны.
Меньше мелких кусочков, больше 1-2 крупных форм + пара мелких рядом.
**Приёмка:** узлы не путаются с декором; вид от amount меняется (кучка тает).

### З4. Композиция лагеря
`entrance.ts`: костёр в центре утоптанного круга (кольцо камушков вокруг огня),
рядом тотем-идол (высокий столб с маской, в цветах племени), кучи ресурсов дугой
вокруг. Фигуры королевской пары оставить. Усилить огонь: в `scene.ts` →
`updateFireGlow` добавить 2-3 «языка пламени» (треугольники со сменой по тику).
**Приёмка:** лагерь читается как сердце кадра, как в макете; Render ms не вырос.

### З5. Пруд с берегом
`ground.ts` (`drawPond` + при желании water-тайлы из
`client/public/assets/environment/summer-plains/`): органичная форма (волны по углу,
как у полян — см. docs/Метод переходов поверхности.md), песчаная кайма берега,
пиксельная бахрома, 2-3 блика. Вода остаётся чисто визуальной (сервер не знает).
**Приёмка:** пруд не выглядит наклейкой; берег «рыхлый», не ровный эллипс.

### З6. Пергаментная тема UI
`main.ts` (только CSS в `style.textContent`): все панели — бежевый пергамент
(#f0e2c0-подобный), коричневые рамки 2-3px, тёмно-коричневый текст, кнопки-плашки;
активная кнопка — тёплый акцент. Тёмную тему не оставлять кусками.
**Приёмка:** все панели (верх, ресурс-бар, задачи, HUD, юнит, минимапа, статус)
в одном стиле; текст читается; ничего не наезжает при 1280px и 1920px ширины.

### З7. Карточки построек
`main.ts`: инструменты «Хижина/Склад/Стена» превратить в карточки нижнего ряда
(по центру снизу, как в макете): иконка (из `spriteIconDataUrl` или мини-canvas),
название, цена. Карточка серая/полупрозрачная, если ресурсов племени A не хватает
(сравнивай с `latestWorld.colonies[0].colony`). «Еда/Зоны/Ластик» оставить сверху.
Логику отправки команд не менять — только внешний вид и место кнопок.
**Приёмка:** цена видна до клика; при нехватке глины карточка «Стена» серая, но
клик по серой карточке не ломает игру (можно просто не выбирать её).

### З8. Спрайты ролей
`sprites.ts` + `entities.ts` (`updateSurfaceAnts`): варианты человечка по занятию:
- `job === "guard"` — копьё в руке (убрать мою временную «копьё-черту» из Graphics);
- `job === "build"` — молоточек;
- несёт груз — цвет груза по `carryKind`: еда жёлтая, глина рыжая, дерево коричневое,
  камень серое (сейчас всегда жёлтый кусок в `clayfolkCarry`).
Палитры обоих племён (dark/red) сохранить.
**Приёмка:** в толпе с одного взгляда видно стража, строителя и ЧТО несёт носильщик.

### З9. Микро-анимации
`entities.ts` (+ при необходимости пара кадров в `sprites.ts`): дешёвые 2-кадровые
эффекты по `world.tick`: у добывающего (`job harvest` рядом с узлом) — лёгкий
«клевок» к узлу; у строителя у площадки — покачивание; у всех при ходьбе уже есть
наклон — не сломать. Без спрайт-листов и таймлайнов, только смена кадра/смещение.
**Приёмка:** толпа «суетится»; на скорости 20x не мельтешит до ряби; Render ms ок.

### З10. Полировка каркасов Fable 5
`main.ts`: причесать под тему З6 мои черновики — ресурс-бар (иконки/отступы),
панель «Задачи» (зелёная галка вместо перечёркивания — по вкусу Степана), панель
юнита (портрет жителя слева — мини-canvas со спрайтом), минимапа (рамка-паспарту).
Логику (id элементов, обновление по снапшоту, клики) НЕ менять.
**Приёмка:** Степан говорит «выглядит как одна игра, а не два стиля».

### З11. Масштаб и объём построек (просьба Гриши)
`scene.ts` → `updateBuildings`: постройки должны соответствовать размеру жителей
(спрайт жителя ~54px на зуме 1):
- **стена**: выше жителя — рисуй с «объёмом»: тёмная передняя грань (низ), светлая
  верхняя грань, высота над землёй ~1.5 клетки; соседние сегменты сливаются в
  сплошную стену (без швов между клетками одного ряда);
- **хижина**: радиус ~3.5-4 клетки (сейчас 2.1), купол с бликом, тёмный дверной проём;
- **склад**: ~3 клетки, «сарай» с крышей.
Сервер (коллизии, клетки 2x2) не трогать — это только отрисовка.
**Приёмка:** житель рядом со стеной НИЖЕ стены; хижина выглядит домом, а не кнопкой.
