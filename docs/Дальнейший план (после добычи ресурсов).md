## Правила (обновлены с учётом реальности)

1. После КАЖДОГО серверного изменения: `npm run check` + `npm run check:slice`.
2. Смена `shared/types.ts` = bump CURRENT_SNAPSHOT_VERSION и
   CURRENT_PROTOCOL_VERSION = Сброс. Это штатно.
3. Старое правило «Codex не пускать в server/src/sim/» де-факто пересмотрено:
   добычу Codex сделал в sim/ сам и сделал хорошо. Новое правило: Codex МОЖНО
   в сервер, но малыми шагами, с обязательными проверками из п.1 и отдельным
   коммитом на каждый шаг, чтобы легко откатить.
4. Маленькие коммиты; main всегда рабочий; незакоммиченного не копить (урок
   этапа 1: месячная работа висела в рабочей папке).
5. Баланс/поведение смотреть с чистого мира (Сброс.bat).
6. Вести чеклисты в docs как раньше: отметка в том же коммите, что и код.
