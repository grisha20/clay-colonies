# ТЗ: угловые тайлы берега озёр Clayfolk

## Задача

Создать пиксельный autotile-атлас углов берега для больших озёр Clayfolk. Углы должны бесшовно продолжать существующую прямую береговую текстуру Summer Plains.

Исходный референс в проекте:

```text
client/public/assets/environment/summer-plains-water-animation-demo/
  water_animation_demo/water_animation_demo.png
```

Размер исходного листа: `192×128 px`. Главный образец берега — левый фрагмент `x=0, y=0, width=32, height=64`:

- верхние `32 px`: трава, жёлто-песчаная кромка и серо-зелёные камни;
- нижние `32 px`: яркое бирюзовое мелководье;
- пиксельная сетка: `32×32 px`;
- фильтрация: только nearest-neighbor, без сглаживания.

## Требуемый результат

Один PNG-файл:

```text
lake_shore_corners.png
```

Параметры:

- размер листа: `256×128 px`;
- восемь тайлов по `64×64 px`;
- прозрачный фон вне берегового перехода;
- формат RGBA PNG;
- без теней за пределами тайла;
- без полупрозрачных размытых пикселей;
- строго пиксельная графика;
- без текста, рамок, подписей и промежутков между тайлами.

## Раскладка атласа

```text
y=0..63
[ внешний NW ][ внешний NE ][ внешний SW ][ внешний SE ]

y=64..127
[ внутренний NW ][ внутренний NE ][ внутренний SW ][ внутренний SE ]
```

Координаты:

| Тайл | X | Y | Где вода |
|---|---:|---:|---|
| Внешний NW | 0 | 0 | справа-снизу |
| Внешний NE | 64 | 0 | слева-снизу |
| Внешний SW | 128 | 0 | справа-сверху |
| Внешний SE | 192 | 0 | слева-сверху |
| Внутренний NW | 0 | 64 | везде, кроме левого верхнего квадранта |
| Внутренний NE | 64 | 64 | везде, кроме правого верхнего квадранта |
| Внутренний SW | 128 | 64 | везде, кроме левого нижнего квадранта |
| Внутренний SE | 192 | 64 | везде, кроме правого нижнего квадранта |

`NW/NE/SW/SE` обозначает положение угла водоёма. Внешний угол содержит один водный квадрант. Внутренний угол содержит три водных квадранта.

## Художественные требования

Порядок перехода от суши к воде должен совпадать с референсом:

```text
зелёная трава
→ тонкая жёлто-песчаная полоса
→ неровный ряд серо-зелёных камней
→ светлая бирюзовая вода
```

Требования к рисунку:

- палитру брать только из приложенного референса;
- размер и плотность камней сохранять как на прямом берегу;
- угол проходит через центр тайла `(32, 32)`;
- на середине каждой стороны рисунок должен точно переходить в соответствующую прямую полосу шириной `32 px`;
- не добавлять растения, брёвна, ракушки, пену, здания и персонажей;
- не создавать круглые острова или готовое озеро;
- не менять рисунок волн и масштаб пикселей;
- внешние углы могут быть слегка срезаны на 1–3 пикселя, но общий силуэт остаётся прямоугольным;
- внутренние углы должны быть читаемыми и не содержать чёрных дыр или наложенных каменных комков.

## Технические швы

У каждого тайла проверить четыре контрольные линии:

- `x=0`;
- `x=63`;
- `y=0`;
- `y=63`.

На стороне, где продолжается прямой берег, последние пиксели обязаны совпадать с соответствующим повёрнутым фрагментом исходной текстуры `32×64 px`. При укладке нескольких одинаковых углов и прямых тайлов не должно быть:

- разрыва песчаной полосы;
- двойного ряда камней;
- бирюзового или тёмного квадрата на траве;
- скачка толщины мелководья;
- полупрозрачной линии;
- изменения размера пикселей.

## Критерии приёмки

1. Все восемь тайлов находятся на точных координатах атласа.
2. Размер каждого тайла ровно `64×64 px`.
3. Внешние и внутренние углы визуально различаются.
4. Каждый угол стыкуется с прямой полосой без заметного шва.
5. Атлас сохраняет исходную палитру Summer Plains.
6. При масштабе 800% нет сглаживания и новых промежуточных цветов на границах пикселей.
7. Вокруг тайлов нет цветной подложки.
8. PNG открывается с прозрачностью и не содержит лишних полей.

## Готовый промпт для генератора

```text
Используй приложенное изображение как обязательный визуальный и технический референс. Создай pixel-art sprite sheet с углами берега для 2D top-down игры.

Нужен один RGBA PNG размером ровно 256×128 пикселей. Внутри восемь тайлов 64×64 без промежутков и подписей.

Верхний ряд: четыре внешних угла водоёма — NW, NE, SW, SE.
Нижний ряд: четыре внутренних вогнутых угла — NW, NE, SW, SE.

Стиль и палитра должны точно совпадать с приложенной прямой береговой текстурой: зелёная трава, тонкая жёлто-песчаная кромка, серо-зелёные камни, яркое бирюзовое мелководье. Сохрани исходный размер пикселей и толщину каждого слоя.

Каждый угол проходит через центр тайла 32,32. К середине каждой стороны угол обязан переходить в прямой берег шириной 32 пикселя. Внешний NW: вода справа-снизу. Внешний NE: вода слева-снизу. Внешний SW: вода справа-сверху. Внешний SE: вода слева-сверху. Для внутренних углов вода занимает три квадранта, а суша — квадрант, соответствующий названию.

Прозрачный фон вне берегового перехода. Никакого antialiasing, blur, света, объёмных эффектов, текста, рамок, персонажей, растений или дополнительных объектов. Не рисуй готовое озеро. Не меняй масштаб волн. Не добавляй тёмную или бирюзовую квадратную подложку вокруг углов.

Главный приоритет: бесшовное соединение с приложенной прямой текстурой при поворотах на 90 градусов.
```

## Что приложить к запросу

Приложить исходный файл `water_animation_demo.png` целиком. Дополнительно указать генератору, что главным эталоном является его левый фрагмент `32×64 px`.
