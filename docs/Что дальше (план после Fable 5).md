## 0. Сначала про склад (почему «всё лежит на виду»)

Сдача ресурсов В МЕХАНИКЕ работает: носильщик идёт в ближайшую точку (вход ИЛИ
достроенный склад) — проверено headless-тестами (36-56 сдач у склада за прогон).
Но ВИЗУАЛЬНО кучи еды/глины/дерева/камня рисуются ТОЛЬКО у костра
(`client/src/render/surface/entrance.ts`), потому что запас племени — общее число,
а не «содержимое конкретного здания». Поэтому кажется, что в склад никто ничего
не носит: житель подошёл к складу, груз исчез, куча выросла… у костра.

**Мелкая задача-фикс (кому угодно):** рисовать часть куч у склада. Простой способ:
в `entrance.ts` / `scene.ts` при отрисовке достроенного склада рисовать рядом с ним
кучки пропорционально общему запасу (например 30% визуального объёма), а у костра —
остальное. Механику не трогать. Плюс, если хочется честности по-настоящему —
это отдельное серверное решение «запас хранится в зданиях» (см. п.3, не раньше Фазы 5).

построек над стенами. Снапшот v5.

**Незакоммиченное:** у Степана в рабочей папке правки рендера (ground/entities/
environment/sprites/scene — включая мой З11: крупные объёмные постройки).
Степан: запусти `npm run check`, посмотри глазами и закоммить со своей стороны.

## 2. Ближайшая очередь (по убыванию важности)

1. **Ворота в стене** (сервер, ~1 вечер). Игроки строят замкнутые заборы — жители
   выбираются только «порывом». Нужен сегмент «ворота»: как стена, но свои жители
   проходят (паук и чужие — нет). Реализация: `BuildingType "gate"`, в
   `resolveWallCollision`/`isWallBlockedAt` — второй Set `gateCells`, проверка
   colonyId агента. Команда: инструмент «Ворота» или клик ластиком по своей стене
   превращает сегмент в ворота. Снапшот v6 + Сброс.
2. **Панель приоритетов работ** (Фаза 5, сервер+клиент). Гриша просил: «таскают
   только еду». Ползунки/кнопки: еда / стройка / добыча / оборона -> целевые числа
   ролей в `economy.ts` (замена констант maxHarvesters/maxActiveBuilders/guardCount).
   Команда `setPriorities`, хранить в Colony (v6).
3. **Кучи у склада** (визуал, см. п.0).
4. **Очередь Codex** (по ТЗ, файлы сейчас у Степана в работе): З1 палитра,
   З3 крупные узлы ресурсов, З4 лагерь с тотемом, З5 берег пруда, З8 спрайты ролей
   (страж/строитель/груз по типу), З9 микро-анимации, З10 полировка панелей.
5. **Известные мелочи:** зажатый между готовыми стенами сегмент может не
   достроиться (строителю некуда встать) — лечится воротами/сносом; зона запрета
   мягкая (~1% заходов на кромке); у носильщика не видно ТИП груза (это З8).

## 4. Как работать без Fable 5 (правила самопроверки)

- После КАЖДОГО серверного изменения: `npm run check` и `npm run check:slice`.
- Меняешь баланс/поведение — смотри с чистого мира (Сброс.bat).
- Меняешь `shared/types.ts` — подними CURRENT_SNAPSHOT_VERSION и
  CURRENT_PROTOCOL_VERSION (старые сохранения отбросятся сами — это штатно).
- Codex НЕ пускать в `server/src/sim/`, `shared/types.ts` и версии снапшота.
- Маленькие коммиты; main всегда рабочий.

## 5. Когда вернётся Fable 5 — что дать ему первым

1. Скриншот текущего вида (как в этот раз) — быстрее всего синхронизирует.
2. Список «что раздражает глазами» от Гриши — как в этот раз, это работает.
3. Незакоммиченные файлы к тому моменту закоммитить — чтобы вся картина была в git.
