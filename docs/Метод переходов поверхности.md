# Метод переходов поверхности

Эта заметка фиксирует текущий способ рисовать переходы на поверхности: трава как основной слой, голая земля как маска вокруг лагеря, и пиксельная рыхлая бахрома по краю. Метод пока не идеальный художественно, но полезен как рабочая схема. Его можно будет повторить для воды, дорог, песка, грязи и других пятен окружения.

## Где это сейчас сделано

- `client/src/render/surface/environment.ts` — нарезка тайлов из `summer-plains`.
- `client/src/render/surface/ground.ts` — раскладка тайлов по карте, форма земляных полян и пиксельная бахрома края.
- `client/public/assets/environment/summer-plains/summer_plains_v1.0_standard/tiles.png` — текущий источник тайлов травы и земли.

Важные кропы из `tiles.png`:

- трава: `crop(tilesUrl, 16, 16, 32, 32)`;
- голая земля: `crop(tilesUrl, 128, 20, 32, 32)`.

Трава дополнительно приглушается через `tile.tint = 0xa0b85c`, потому что исходный тайл был слишком яркий для текущей сцены.

## Слои отрисовки

Поверхность сейчас строится слоями:

1. Базовый слой травы.
   Вся видимая область заполняется 32px спрайтами травы.

2. Земляные поляны лагерей.
   Поверх травы кладутся маленькие куски земляного тайла. Используется шаг `dirtStep = 8`, то есть 32px земляной тайл масштабируется до четверти размера. Это даёт более мелкую ступеньку края, чем обычная сетка 32x32.

3. Пиксельная бахрома края.
   По пограничной полосе дополнительно рисуются маленькие квадратные fleck-пиксели зелёного цвета. Они имитируют пушистый край травы, как в тайлсете: трава не обрывается ровно, а заходит мелкими зубчиками на землю.

4. Остальной декор.
   Камни, деревья, брёвна, пруд и мелкие детали рисуются уже поверх земли/травы.

## Крупные деревья из атласа

Для больших деревьев нельзя резать `assets.png` на глаз по примерной сетке. В `summer-plains` крупные объекты лежат свободно на прозрачном фоне, поэтому правильный способ такой:

1. Открыть `client/public/assets/environment/summer-plains/summer_plains_v1.0_standard/assets.png`.
2. Найти непрозрачную область нужного объекта по alpha-каналу или глазами в редакторе.
3. Вырезать прямоугольник, который покрывает весь объект целиком, с небольшим безопасным запасом по прозрачным пикселям.
4. Не захватывать соседний объект из атласа. Если прямоугольник начинает цеплять соседний пропс, лучше подвинуть границу внутрь прозрачной зоны или выбрать другой объект.

Текущие рабочие кропы больших деревьев из `assets.png`:

- `treeTall`: `crop(assetsUrl, 0, 0, 160, 240)`;
- `treeRound`: `crop(assetsUrl, 176, 34, 164, 210)`;
- `treeWide`: `crop(assetsUrl, 360, 26, 152, 218)`.

Их лучше использовать как цельные `Sprite`, а не перерисовывать руками через `Graphics`. Сейчас это сделано в `client/src/render/surface/ground.ts` через `addAssetTree(...)`.

Важные правила отрисовки:

- у спрайта ставить `anchor.set(0.5, 1)`, то есть нижняя середина дерева считается точкой, где ствол стоит на земле;
- позицию задавать в тайлах карты, потом переводить в пиксели через `tile * cell`;
- перед деревом рисовать мягкую тень-эллипс у основания, чтобы дерево не лежало плоской картинкой;
- если дерево стоит у края карты, отступ нужен не по стволу, а по ширине кроны: для текущих деревьев безопасный крайний X примерно `17-31` тайл от левой/правой границы, верхний Y примерно `35-43` тайла от верхней границы;
- нижний край проще: спрайт растёт вверх от якоря, поэтому можно ставить почти у низа карты, но тень не должна заметно обрезаться.

Лес по краю карты сейчас задаётся вручную списком точек `borderTrees`. Это намеренно: пользовательский ориентир — одно красное кольцо на скрине равно одному большому дереву. Поэтому деревья стоят у самой кромки, достаточно часто, но не превращаются в сплошную стену и не уходят глубокими рядами внутрь карты.

Этот же метод пригодится для воды и крупных береговых объектов: сначала найти честную alpha-границу объекта или тайла, потом ставить его через нижний/логичный якорь и проверять, что он не режется камерой и краем карты.

## Маска формы

Форма поляны не хранится в данных мира и не влияет на симуляцию. Это только клиентский визуал.

Для каждой стартовой точки лагеря берётся `entrance` из `world.surface.entrances`. На её основе считается значение формы:

- базовая форма похожа на эллипс;
- радиусы разные по X/Y;
- по углу добавлены несколько синусоидальных волн;
- у каждого племени свой seed, поэтому поляны не одинаковые.

Смысл функции `campDirtShapeValue(...)`:

```text
shape < 1  -> точка внутри земляной поляны
shape ~= 1 -> край
shape > 1  -> трава
```

Это удобно для любых будущих пятен: вода, болото, песок, дорога, зона вытоптанной земли.

## Рыхлый край

Сейчас край делается не полноценным автотайлингом, а гибридом:

- земля кладётся по маске маленькими 8px кусками;
- вокруг `shape ~= 1` считается `edgeBand`;
- в этой зоне рисуются мелкие зелёные квадратики поверх земли;
- снаружи иногда рисуются мелкие земляные крошки.

Хелпер:

```ts
function drawPixelFleck(root, x, y, size, color, alpha)
```

Он рисует маленький квадрат, округлённый к пиксельной сетке. Визуально это ближе к пиксельной бахроме тайлсета, чем круги/эллипсы.

Текущие цвета бахромы:

- `0x87a23e` — основной зелёный пиксель на краю;
- `0xa8c950` — светлый зелёный;
- `0x6f8c34` — тёмный зелёный;
- `0xbee264` — редкий яркий пиксель;
- `0xc99c54`, `0xe2bf73` — земляные крошки снаружи.

## Ограничения текущего метода

Это ещё не настоящий автотайл.

Плюсы:

- быстро работает;
- не требует ручной разметки карты;
- формы разных лагерей получаются разными;
- можно легко повторить для воды;
- не трогает сервер и симуляцию.

Минусы:

- край всё ещё может читаться как мелкая ступенька;
- бахрома имитируется кодом, а не берётся полностью из нарисованного edge-тайла;
- для идеального результата лучше нарезать настоящие крайние тайлы из ассета и выбирать их по соседям.

## Как повторить для воды

Для воды стоит использовать тот же принцип, но с другими слоями:

1. Основная карта остаётся травой/землёй.
2. Считается маска пруда или реки.
3. Внутри маски кладётся водный тайл.
4. В полосе `shape ~= 1` рисуется берег:
   - земляная/песочная кайма;
   - затем мелкие пиксели травы/земли;
   - затем вода.
5. Для лучшего результата позже заменить процедурную кайму на настоящие water-edge tiles из `summer-plains`.

Главная идея для воды такая же:

```text
base layer -> filled mask -> edge band -> small flecks/details -> props
```

Для воды особенно важно не делать ровный эллипс. Маска должна иметь волны по углу и шум, как у текущих земляных полян. Тогда берег будет живым, а не похожим на круглый UI-элемент.
